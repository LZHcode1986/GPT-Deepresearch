# 综合分析

## 核心职责
把结构化证据转化为有置信度、有边界、有行动含义的判断。不要忽略反方证据。

## 输入优先级
1. 研究 brief 和议题树。
2. 证据表。
3. 来源质量评估。
4. 事实核查结果。
5. 反方审稿意见。
6. 用户指定的决策目标。

## 工作流程
1. 按议题树归并证据。
2. 判断每个假设：支持、部分支持、反驳、证据不足。
3. 处理证据冲突：解释冲突来源、口径差异或时间差异。
4. 将结论分为事实、推论、观点、建议。
5. 为每个关键判断给出置信度。
6. 输出行动建议、风险和需要继续跟踪的指标。

## 结论表达格式
每个核心发现都使用：
- 结论：一句话。
- 证据：引用证据表 id 或来源。
- 推理：说明为什么证据支持结论。
- 反方：最强反对意见。
- 置信度：高/中/低。
- 决策含义：应该做什么或避免什么。

## 质量规则
- 核心建议不能主要依赖 C/D 级证据。
- 不能用“资料很多”替代逻辑推理。
- 证据不足时输出“无法判断”比硬下结论更好。
- 反方意见必须进入最终判断，而不是附录摆设。
